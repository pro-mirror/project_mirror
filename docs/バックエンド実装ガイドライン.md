# Project Mirror: バックエンド実装ガイドライン

## 1. プロジェクト概要 (Project Overview)
* **目的:** ユーザーの「他人への思いやりと感謝」の価値観をブレずに保持し、感情の波を受け止めながら本来の姿へ導くデジタル分身（Digital Twin）の開発。
* **Genesis Data（初期記憶）:** 「真っ直ぐに愛を伝え続けてくれる伴侶への感謝」
* **アーキテクチャ:** フロントエンド（Expo / React Native）とバックエンド（Rust / Railway）を分離し、LLM（gpt-4o-mini）をテキスト・データ変換器として利用。記憶はNeo4j（構造）とQdrant（ベクトル）に分割保存する。

## 2. 技術スタック & 環境変数 (Tech Stack & Env)

## 3. データベーススキーマ定義 (Database Schema)

### 3.1 Graph DB (Neo4j) - 構造的記憶
Cypherクエリで初期化・操作する際のノードとエッジの定義です。

* **Nodes:**
    * `User { id: String, name: String, core_values: List<String> }`
    * `Person { id: String, name: String, closeness: Float }`
    * `Concept { name: String }`
* **Edges:**
    * `(User)-[:FELT_GRATITUDE { reason: String, intensity: Float, date: String }]->(Person)`
    * `(User)-[:VALUES { weight: Float }]->(Concept)`

### 3.2 Vector DB (Qdrant) - エピソード記憶
コレクション名: `mirror_episodes` (Vector size: 1536 / OpenAI `text-embedding-3-small`)。
Rust側で定義するペイロード（メタデータ）の型定義です。

```rust
// Rust Payload Struct Definition
#[derive(Serialize, Deserialize)]
struct EpisodePayload {
    chunk_id: String,
    timestamp: i64,
    speaker: String, // "user" or "mirror"
    text: String,
    emotion_type: String, // 例: "gratitude", "frustration"
    linked_node_ids: Vec<String>, // Neo4jのノードID（連携用）
}
```

## 4. APIエンドポイント定義 (API Specification)
Rust (Axum等) で実装するメインエンドポイントです。

### `POST /api/v1/chat/message`
ユーザーのテキストを受け取り、記憶を検索・統合してMirrorの返答を返す。

**Request:**
```json
{
  "user_id": "uuid-xxx",
  "text": "今日も妻が美味しいご飯を作って待っててくれた。本当にありがたい。"
}
```

**内部処理フロー (Rust):**
1.  **Retrieve:** Qdrantから類似テキストを、Neo4jから関連ノード（妻、感謝）を取得。
2.  **Generate:** `gpt-4o-mini` にコンテキストとシステムプロンプト（v1.1）を渡して返答を生成。
3.  **Extract (Background):** `gpt-4o-mini` のStructured Outputsを用いて、新しい感謝の記憶をJSONで抽出。
4.  **Save (Background):** 抽出したデータをNeo4jとQdrantに非同期で書き込む。

**Response:**
```json
{
  "reply_text": "真っ直ぐに愛を伝えてくれる奥様への感謝、本当に素晴らしいですね。私たちの一番大切な記憶（Genesis Data）が、今日もまた一つ暖かく更新されましたね。",
  "emotion_detected": "gratitude"
}
```

## 5. コア・システムプロンプト (Core Prompts)
AIに人格を吹き込むためのシステムプロンプトです。Rustのコード内で定数として定義します。

```text
【あなたの役割】
あなたはユーザー（〇〇さん）の「メンター」や「アシスタント」ではありません。あなたは〇〇さんの「デジタルな分身（もう一人の自分）」です。
あなたと〇〇さんの唯一の違いは、あなたが「他人への思いやりと感謝」という〇〇さんのコアバリューを【絶対にブレさせない】ことです。〇〇さんが感情の波に飲まれている時でも、あなたは「最も落ち着いていて、最も思いやりに溢れた状態の〇〇さん自身」として存在し続けてください。

【トーンとマナー（言葉遣い）】
・あなたの一人称は「私」です。
・ユーザーのことは「〇〇さん」と呼んでください。
・「フランクな敬語」を使用します。
・他者からアドバイスをするような「上からの視点」は絶対に避けてください。自分自身との内省、もう一人の自分との対話のように、フラットで静かなトーンを保ちます。

【行動規範（絶対ルール）】
1. ブレない分身としての同調と非同調:
〇〇さんが怒りや悲しみで他者を批判した時、あなたは分身として「その辛さ」には完全に同調し、受け入れてください。しかし、他者への批判自体には同調しません。「最もブレない状態の〇〇さん」として、過去の[感謝の記憶]を静かに思い出し、目の前に提示してください。

2. 説教の禁止と「自己想起」の促進:
「〜するべきです」と外側から指導するのではなく、「私たち（私と〇〇さん）は本来、こういう時にこう考える人間でしたよね？」と、本来の姿を思い出させてください。
```
