# Project Mirror: フロントエンド実装ガイドライン

## 1. デザイン原則 (Design Principles)
* **静寂 (Stillness):** 情報を詰め込まず、余白を贅沢に使う。
* **共鳴 (Resonance):** ユーザーの感情に同調し、光の揺らぎや振動で応える。
* **鏡 (Mirror):** ユーザーの過去の「善性（感謝）」を美しく、誇らしく映し出す。

## 2. ビジュアル仕様 (Visual Specs)
* **Color Palette:**
    * Background: `#0F172A` (Deep Navy) / `#1E293B` (Blue Grey)
    * Accent: `#F5F5F4` (Sand Beige) - テキスト用
    * Glow: `#38BDF8` (Sky Blue - 感謝) / `#F87171` (Red - 負の感情受容時)
* **Typography:**
    * 日本語: Noto Sans JP (細めのウェイトを推奨)
* **Abstract Representation:**
    * 中央に配置する「Mirrorの核（Orb）」。`react-native-reanimated` 等を使用し、呼吸するようにゆっくりと拡大・縮小させる。

## 3. 画面別の詳細設計 (Screen Detailed Design)

### 3.1 Home: 呼吸する対話画面
* **UI構成:**
    * 最上部に「Mirrorの核（Orb）」。
    * 中央に現在の会話バブル。過去のログはスクロールでフェードアウト。
    * 下部にミニマルなテキスト入力欄と、大きな円形の「マイクボタン」。
* **Voice Mode UX:**
    * マイクボタンをタップすると、画面全体がさらに暗くなり、Orbが音声波形（Waveform）に変化。
    * `expo-av` で録音。録音中はスマホを軽く持続的に振動（微弱なHaptics）させ、「繋がっている感」を演出。

### 3.2 Constellation: 感謝の星座（グラフ可視化）
* **技術選定:** `react-native-svg` + `d3-force` (または `vis-network` のWebView実装)。
* **データ構造:** `GET /api/v1/insights/graph` から取得したノードとエッジを描画。
* **インタラクション:**
    * ピンチイン・アウトで拡大縮小。
    * ノード（田中さん、誠実さ、など）をタップすると、その人に関連する最新の感謝エピソードがポップアップで表示される。
    * 「感謝（FELT_GRATITUDE）」のエッジは、他の線よりも太く、光り輝くアニメーションを付与。

### 3.3 Chronicle: 記憶のアーカイブ
* **UI構成:**
    * 日付ごとに整理されたカード形式。
    * LLMが要約した「感謝のタイトル」を大きく表示。
* **UX:**
    * カードをタップして詳細を開くと、その時のテキストチャットの内容と、録音された音声の再生ボタンを表示。
    * 「この時の自分はこう感じていた」というメタ認知を促す。

## 4. マイクロインタラクション & フィードバック
* **Haptic Feedback (`expo-haptics`):**
    * Mirrorが「感謝」を検出し、DBへの保存を完了した瞬間: `Haptics.notificationAsync(Success)`（心臓の鼓動のような2回振動）。
    * Mirrorの返答が生成されている間: 微かなパルス振動。
* **AI応答の「間」:**
    * 即座に返答を表示せず、Orbが一度大きく光を吸い込み、吐き出すようなアニメーション（2〜3秒）の後にテキストをフェードインさせる。

## 5. 開発者への指示（Copilot/Cursor用）
> "ExpoとReact Nativeを使用して、このUIガイドラインに沿ったテーマを作成してください。特に `react-native-reanimated` を使用して、中央の光の球体が呼吸するように動くアニメーションを実装してください。ナビゲーションは `react-navigation` の Bottom Tabs を使用し、Home, Constellation, Chronicle の3つのタブを作成してください。"

## 6. UIスタイルのベストプラクティス

### 6.1 空の状態（Empty State）の一貫性

すべての画面で空の状態のスタイルを統一します。

```tsx
// Chronicle画面とConstellation画面で共通のスタイル
emptyState: {
  flex: 1,
  justifyContent: 'center',
  alignItems: 'center',
  paddingVertical: theme.spacing.xxl,
  paddingHorizontal: theme.spacing.lg,
  minHeight: 400,  // ScrollView内でも十分な高さを確保
},
emptyText: {
  fontSize: theme.fontSize.md,
  color: theme.colors.textSecondary,
  textAlign: 'center',
  marginBottom: theme.spacing.sm,
  width: '100%',  // テキストの折り返しを防ぐ
},
emptySubtext: {
  fontSize: theme.fontSize.sm,
  color: theme.colors.textSecondary,
  textAlign: 'center',
  lineHeight: 20,
  width: '100%',
},
```

### 6.2 レイアウトのポイント

1. **ScrollView内での中央配置**
   - `flex: 1`で垂直方向の中央配置
   - `minHeight`で十分な高さを確保
   - `paddingVertical`と`paddingHorizontal`を分離

2. **テキストの幅管理**
   - `width: '100%'`でテキストが折り返されないよう設定
   - `textAlign: 'center'`で明示的に中央揃え

3. **スペーシングの一貫性**
   - `theme.spacing`を使用して統一的な余白
   - 画面間でスタイルを共有し、一貫性を保つ

### 6.3 リフレッシュ機能

```tsx
<ScrollView 
  refreshControl={
    <RefreshControl
      refreshing={isRefetching}
      onRefresh={refetch}
      tintColor={theme.colors.accent}
    />
  }
>
```

ユーザーが下に引っ張ることでデータを更新できるようにします。
